What are you building, and who is it for?

Orquestra turns Solana program IDLs into instant APIs and AI-native MCP tools. Upload an Anchor or Codama IDL and automatically get hosted REST APIs, generated docs, transaction builders, PDA derivation, account decoding, simulation tools, and structured MCP access for AI agents.

It is built for Solana protocol teams that want easier integrations, backend and fintech developers who prefer REST APIs over SDK complexity, and AI agents that need structured, safe access to on-chain programs.

The goal is simple: make Solana programs accessible from any programming language, application, automation workflow, or AI system without requiring custom backend infra

It’s for developers: integrate any Solana program from any language without SDK complexity. It’s for AI agents: structured MCP tools for safe on-chain interactions. It’s for protocols: instant API and It’s for users: everything becomes easier to access and use. It’s for protocols: instant API and AI compatibility for their programs.

Why did you decide to build this, and why build it now?

After four years of building and deploying Solana programs, the same problem kept appearing: every integration required repetitive backend work. SDK setup, transaction wrappers, discriminator handling, and outdated documentation. Solana scaled fast, but accessibility did not.

We scanned more than 52,000 on-chain programs. Only 4K+ had accessible IDLs . Thousands of programs exist today that developers and AI agents still cannot interact with programmatically.

At the same time, AI agents are becoming a real developer interface through MCP standards introduced by Anthropic and adopted across the industry. Agents need structured access to Solana programs, not raw RPC complexity.

Orquestra solves this by transforming any Solana IDL into instant APIs and AI-native MCP tools, making programs understandable, callable, and automatable for developers, applications, and AI agents.

The AI age is here. Solana needs a better interface layer.

Access instructions

Judges can access the platform directly using GitHub authentication. They can either upload a Solana program IDL manually or import it automatically from a program URL.

Important GitHub repo context

The main product consists of multiple repositories and services working together as the Orquestra ecosystem.

https://github.com/berkayoztunc/orquestra-cli contains the CLI tool used for transaction building, signing flows, and local developer integration. https://github.com/berkayoztunc/orquestra-signer-mcp contains the MCP-based signer implementation built on top of Solana Keychain concepts for AI agent signing workflows.
